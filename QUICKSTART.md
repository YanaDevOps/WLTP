# 🚀 Быстрый старт с WLTP

## ⏱️ Через 10 минут у вас будет готовый portable EXE!

### Шаг 1: Скачайте EXE (автоматически)

1. **Откройте**: https://github.com/YanaDevOps/WLTP/actions
2. **Дождитесь зелёной галочки** ✅ рядом с названием build
3. **Прокрутите вниз** до раздела **"Artifacts"**
4. **Нажмите** на `WLTP-Windows-Portable.zip`
5. **Распакуйте** ZIP файл в любую папку

### Шаг 2: Запустите WLTP

**Важно:** Запустите от имени **Administrator** (ПКМ → "Запуск от имени администратора")

Причина: нужна raw ICMP sockets для traceroute

### Шаг 3: Используйте!

1. В поле ввода введите хост:
   - `google.com`
   - `8.8.8.8`
   - `yoursite.com`

2. Нажмите **"Start Trace"**

3. Смотрите результаты в реальном времени:
   - 🟢 Зелёный = всё хорошо
   - 🟡 Жёлтый = внимание
   - 🔴 Красный = проблема
   - ⚪ Серый = неизвестно (нормально для промежуточных хопов)

### Шаг 4: Понимайте результаты

**Summary Card (вверху):**
- Показывает общее состояние соединения
- Основные находки
- Рекомендации по действиям

**Hops Table (основная таблица):**
- **Hop** - номер хопа
- **Host** - IP или hostname
- **Loss%** - процент потери пакетов
- **Sent/Recv** - отправлено/получено пакетов
- **Best/Avg/Worst** - лучшая/средняя/худшая задержка
- **Jitter** - вариация задержки
- **Interpretation** - умный анализ что происходит

### Шаг 5: Экспорт (если нужно)

**Для отправки в поддержку:**
1. Нажмите **"Export HTML"**
2. Откройте файл в браузере
3. Отправьте по email/в тикет

**Для технического анализа:**
1. Нажмите **"Export JSON"**
2. Откройте в текстовом редакторе
3. Анализируйте структурированные данные

## 🎨 Настройки

Нажмите **"Settings"** чтобы изменить:

- **Theme**: System / Light / Dark
- **Explanation Level**: Simple / Detailed
- **Probe Interval**: интервал между запросами (мс)
- **Max Hops**: максимальное число хопов
- **Timeout**: таймаут на каждый запрос

## ⚠️ Важные нюансы

### Промежуточный хоп не отвечает = НОРМАЛЬНО!

Если видите `***` или `Request timed out` на промежуточном хопе, но следующие хопы отвечают - **это нормально!**

Это означает:
- Роутер deprioritizes ICMP (отдаёт приоритет трафику)
- Или rate-limits ICMP ответы

**WLTP правильно это интерпретирует** и не будет показывать как проблему.

### Потеря пакетов на последнем хопе = ПРОБЛЕМА

Если destination (последний хоп) показывает потерю пакетов - **это реальная проблема!**

Это означает:
- Сервер перегружен
- Проблемы с сетью
- Блокировка ICMP

### Высокий jitter = проблемы с VoIP/Играми

Jitter > 50ms может вызывать:
- Прерывания в VoIP звонках
- Лаги в онлайн играх
- Проблемы с видеоконференциями

## 🔧 Troubleshooting

**Ошибка "Permission denied":**
- Запустите от имени Administrator

**Windows SmartScreen блокирует:**
- "Подробнее" → "Выполнить в любом случае"
- Это нормально для unsigned приложений

**Ничего не происходит при нажатии Start:**
- Проверьте что хост введён правильно
- Проверьте интернет соединение
- Попробуйте другой хост (например, 8.8.8.8)
